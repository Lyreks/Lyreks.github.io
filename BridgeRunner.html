BridgeRunner

Made by Skyler Vogt for CSE3541 (Huamin Wang)

Objective

BridgeRunner is a simple Unity game of the endless runner variety in which the player tries to achieve the highest score possible without hitting any obstacles. For added difficulty, the speed of the game will increase every so often.

The player navigates using the keys A or Left Arrow to move left and D or Right Arrow to move right, earning points by maneuvering obstacles successfully. During the game, the score is displayed in the top-center of the screen for those curious eyes to wander.

Collision

The crates have standard box collider components for consistent collision. These are tagged as enemies to ensure they are the only game objects that can end the game with collision.

Each obstacle has a plane trigger that increments the score on passthrough. These are tagged as checkpoints to ensure they are the only game objects that can increment the score.

Continuous Gameplay

As the genre suggests, the game will not end until the player hits an obstacle. The game has a script named TileManager that will continuously load random prefabs every time the player moves about 15 units (the length of the prefabs). These prefabs consist of a bridge, and multiple layouts of crates and checkpoints. I have to credit N3K EN for his tutorial on this method of recurring random generation because it works splendidly.

In order to avoid memory issues, the prefabs are destroyed shortly after they're out of the player's view. This allows the game to truly go on endlessly.

End Screen

Once the game is over, the player's score will be displayed and he or she will be prompted with options to retry or exit. Pressing the retry button will reload the scene, allowing the player to try again. Pressing the exit button will quit the application.

Audio

Using a custom AudioManager script made implementing sound effects and background music very simple. Thank you, Brackeys, for the concise tutorial on how to add these. Utilizing methods from this script, I used a random call similar to the one in the TileManager to make different crash sounds play upon each death.
